Key Terms

Key Terms

  • Git: A software tool that manages version control.
  • Git Repository (Repo): All of the files, folders, and configuration that compose a project managed by Git.
  • GitHub: A web-based platform that stores Git repositories combined with project management and collaboration tools.

Key Terms

  • Clone: Download a repository to your computer for the first time.
  • Fetch/Pull: Get all of the changes to the repository made by others and integrate them into your local copy.
  • Commit: A change to a file in a repository.
  • Commit Message: A short, detailed, "human-readable" summary of the changes made in a commit.
  • Push local commits to the server.

Activity: install GitHub Desktop, create an account, and create a blank repository.

What is Markdown?

  • A text formatting language used in many applications (not just GitHub) for formatting blocks of text for display.
  • In GitHub, Markdown can be used to format the text that displays within the GitHub window

Markdown Basics

  • # Heading Level 1
  • ## Heading Level 2
  • **bold text**
  • *Italics*
  • > blockquote
  • * List Item
  • ```code```
  • [Link](http://www.google.com)
  • Horizontal Rule: Blank Line, ---, blank line
  • ![Image Caption](imageurl imagealttext)

Create a Markdown Page

  • Edit the Readme.md
  • Add your favorite quote, a link to your favorite recipe, your name as a header, and three random facts about yourself. Use sections to make this look like a cohesive document.
  • Commit and push to your GitHub Repository

GitHub Issues, Wikis, and Discussions

  • Collaboration Tools
  • Used to document projects
  • Projects - Kanban Boards
  • Issues can be referenced directly in a commit.

Activity: Create an issue in your repository. Make an edit to your repository files, and close the issue with your commit.

Forking

  • Forks allow you to take an existing repository and make it your own.
  • You can modify and create a derivative work from a repository.
  • Changes can be merged back to the original responsitory, kept separate, or both.

Activity: Fork https://github.com/mjsamberg/ncedconnects-training

Branches

  • Branches are for features. They allow multiple people to work on features simultaneously without bothering each other. Changes can be merged back to the main branch when completed.

Activity: Create a branch in the forked repository. Complete the template. Commit your code. Merge back to the main branch.

Pull Requests

  • Pull requests are used to submit changes from forks back to the main branch.

Activity: Create a pull request for your "About Me" template.

Collaborators and Private Repositories

  • Repositories can be public or private. Collaborators can have access to edit, read, or commit to a repository.

Other Resources

  • GitHub can be used to host webpages. You can use raw HTML or use the Jekyll framework.
  • GitHub Classroom can integrate with an LMS to issue and grade coding assignments.
